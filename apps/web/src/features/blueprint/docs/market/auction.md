| 高开 | 低开 | 平开 | 涨停预判兑现 | 全市场量比 |
|---|---|---|---|---|
| N 支 | N 支 | N 支 | N/M (X%) | X.X |

**涨停预判兑现定义**:
- M = Card 2 "涨跌停预判"维度在 9:20-9:25 期间触发的标的总数

## 三段式时间线指示器 (嵌入表格上方)

从独立的 3 列阶段卡片改为**紧凑时间线**，嵌入表格上方:

`9:15 ━━━ 可撤单 ━━━ 9:20 ━━━ 不可撤单 ━━━ 9:25 撮合`

- 高亮当前阶段 + 倒计时
- hover 某阶段 → tooltip 显示该阶段规则和关键信号
- 原有三阶段详细信号保留在 **expandable 模态框**中

### 三阶段详细信号 (模态框中展示)

**9:15-9:20 可撤单阶段**
- 主力试探盘: 大单挂出后快速撤回, 测试市场反应
- 观察撤单比例: 撤单量/委托量 > 50% 说明试探性质强
- 大单方向: 连续大买单(不撤) 暗示开盘强势意愿
- 涨停试探: 有封涨停意愿的票会在此阶段试探封板量

**9:20-9:25 不可撤单阶段**
- 真实意愿显现: 此阶段委托不可撤回, 挂单=真实意图
- 量价变化: 从 9:20 开始的委托量突变是核心观察点
- 封板确认: 涨停/跌停在此阶段能否稳定封住
- 预期差萌芽: 竞价价偏离昨收 > 2% 值得关注

**9:25 撮合定价**
- 集合竞价成交价确定: 成为今日开盘参考价
- 预期差 = 竞价价 vs 昨收: 偏离度是日内第一个信号
- 竞价成交量: 量大=分歧大, 量小=一致预期
- 未匹配量: 大量买单未成交 → 开盘可能快速拉升

## 数据源分层方案

竞价实时数据是本页的核心依赖。常见平台均**不提供真正的实时竞价推送**——聚宽/米筐是回测平台，提供的是历史归档后的竞价快照；Tushare 只有日线级别；AKShare 本质是爬虫库, 通过 HTTP 轮询可以模拟"准实时"，但有反爬风险和频率限制。

### 四层方案

| 层级 | 数据源 | 延迟 | 覆盖范围 | 成本 | 适用阶段 |
|---|---|---|---|---|---|
| **Tier 1: MVP 冷启动** | AKShare 轮询东财行情 | 1-3s | 自选组 (≤200只) | 免费 | 开发验证 |
| **Tier 1.5: 轻量生产** | XTick API (HTTP+WebSocket) | 亚秒级 | 全市场 | 付费 (未公开) | 个人/小团队 |
| **Tier 2: 生产级** | 券商 L2 行情 (QMT/恒生MATIC) | 亚秒级 | 全市场 | 券商 L2 (几百元/年) | 正式上线 |
| **Tier 3: 机构级** | 行情服务商 (万得/恒生/通达信API) | 毫秒级 | 全市场+逐笔 | 万元/年 | 规模化运营 |

### Tier 1.5: XTick 竞价数据字段

XTick (GitHub: xticktop/DemoXtickJava) 专门提供 A 股实时竞价数据, 比 AKShare 多出关键字段:

| 字段 | 说明 | AKShare 是否有 |
|---|---|---|
| price (虚拟参考价) | 实时推送 | ✅ (polling) |
| jjl (竞价量) | 虚拟匹配量 | ✅ (polling) |
| jje (竞价金额) | 匹配金额 (RMB) | ❌ |
| **nol (未匹配量)** | 未成交量 | ❌ |
| **noe (未匹配金额)** | 未成交金额 | ❌ |
| **trend (方向指示)** | +1=买方偏多, -1=卖方偏多 | ❌ |

- 覆盖沪深京三市, 支持 9:15-9:25 + 14:57-15:00
- 历史竞价数据从 2025.7 起可用 (AKShare 需自建归档)
- WebSocket 推送 (非 HTTP 轮询), 无反爬风险

### Tier 1 降级策略 (重要)

Tier 1 阶段**不可能实现全市场秒级监控** — AKShare 是 HTTP 爬虫, 5000+ 股票逐一轮询需 ~1000s, 并发轮询会触发反爬封禁。

**降级方案** (Tier 1.5/2 接入前):
- **策略 A (关注池模式)**: 仅轮询用户自选股 + 昨日涨停 + 龙虎榜标的 (总计 ≤200 只), 2-3s 刷新
- **策略 B (涨幅榜模式)**: 每 10s 抓取竞价涨幅榜前 50 名, 只看头部异动
- 页面标题在 Tier 1 下应显示"关注池竞价监控"而非"全市场", 升级到 Tier 1.5+ 后切换为"全市场"

### 升级路径
- **Tier 1 限制**: AKShare 是爬虫而非行情API, 高频 polling 可能触发反爬; 全市场4000+股票无法1s全量刷新; 缺少未匹配量和方向指示字段
- **Tier 1.5 优势**: XTick 是正规 API, WebSocket 推送, 全市场覆盖, 额外提供未匹配量+方向指示
- **Tier 2 方案**: QMT (迅投) 已在 Settings/Data Sources 规划了券商对接
- **过渡策略**: 前端代码写统一的行情数据接口 (DataProvider), 底层可切换 AKShare polling / XTick WebSocket / QMT push / 万得 API, 上层组件无感知
- **AG Grid 性能**: Tier 2 接入后每秒 2000+ 条推送, 禁止 setRowData 全量刷新, 必须用 applyTransactionAsync 增量更新 + 前端节流 500ms-1000ms 渲染帧率

### 竞价阶段各数据源实际能力

| 数据字段 | AKShare (T1) | XTick (T1.5) | QMT L2 (T2) | 说明 |
|---|---|---|---|---|
| 虚拟参考价 | ✅ 快照 | ✅ 推送 | ✅ 推送 | 核心字段 |
| 虚拟匹配量 | ✅ 快照 | ✅ 推送 | ✅ 推送 | 核心字段 |
| 未匹配量 | ❌ | ✅ | ✅ | 买卖力量差的关键指标 |
| 方向指示 | ❌ | ✅ (trend) | ✅ | 多空偏向 |
| 买一/卖一 | ✅ | ✅ | ✅ (十档) | T2 额外提供十档 |
| 逐笔委托 | ❌ | ❌ | ✅ (L2) | Tier 2 升级后可启用撤单异动检测 |
| 推送方式 | HTTP 轮询 | WebSocket | callback | T1 受限于轮询 |
| 全市场覆盖 | ≤200只/批 | ✅ 全市场 | ✅ 全市场 | T1 需分批 |

## 4 维度检测

### 维度 1: 大单异动
- **描述**: 单笔委托量或金额超出正常范围，可能是主力建仓/出货信号
- **阈值**: 默认: 单笔 > 50 万股 或 > 500 万元
- **动态阈值**: 按标的流通市值/日均成交量自适应调整。大盘股(沪深300成分)阈值上浮2-3倍，小盘股(市值<50亿)下调至默认值的50%
- **阶段权重**: 全时段有效, 但 9:15-9:20 可撤单阶段信号权重 ×0.5 (大单可能是试探后撤回), 9:20+ 不可撤单阶段权重 ×1.0

### 维度 2: 涨跌停预判
- **描述**: 9:20 后委买/委卖比例极端化，预判封板概率
- **阈值**: 默认: 委买/委卖 > 5:1 (涨停) 或 < 1:5 (跌停)
- **动态阈值**: 涨跌停板幅度不同(主板±10%/创业板科创板±20%/北交所±30%)，预判阈值相应调整
- **阶段权重**: 仅 9:20-9:25 有效 (可撤单阶段的涨跌停信号是噪音)

### 维度 3: 量比突变
- **描述**: 竞价阶段的成交量与近期同期对比，量比异常放大说明有资金异动
- **阈值**: 默认: 竞价量比 > 3.0 (vs 近5日同期均量)
- **动态阈值**: 按板块整体活跃度修正。板块整体量比均值>2.0时，个股触发阈值上调至4.0
- **阶段权重**: 全时段有效, 9:20 后量比发散更有意义

### 维度 4: 开盘偏离检测
- **描述**: 9:30 后首分钟成交均价与 9:25 撮合价的偏离度
- **阈值**: 默认: |9:30-9:31均价 - 9:25撮合价| / 9:25撮合价 > 0.5%
- **动态阈值**: 高波动标的放宽至0.8%，低波动标的收紧至0.3%
- **阶段权重**: 9:31+ (事后验证型信号)

## 机制特点

- 最后 3 分钟撮合，委托不可撤回
- 仅深市 (深主板+创业板)，沪市/北交所无此机制
- 深市收盘价由尾盘竞价决定 (非最后一笔)
- 沪市/北交所标的: 14:57-15:00 仍为连续交易, 显示尾盘成交量/价格变化作为参考 (信号纯度低于深市竞价)

## 交易价值

- 尾盘异动往往暗示次日开盘方向
- 指数成分调仓常在尾盘集中执行 (被动基金/ETF)
- 与日间走势偏离度 > 0.5% 值得关注

---

## 核心分析 1: 尾盘竞价对次日开盘的预判价值

- 统计规律: 尾盘竞价成交量放大 + 价格偏离VWAP > 0.3%时，次日同方向延续概率约62%
- 大单主导的尾盘竞价(单笔>100万)对次日预判价值更高，胜率提升至68%
- 逆势尾盘(日内跌但尾盘竞价翻红)是强信号，次日高开概率约65%
- 需排除特殊日: 指数成分调整日/期权到期日/节假日前一天

---

## 核心分析 2: VWAP 偏离度分析

- 定义: 偏离度 = (尾盘竞价价 - 日内VWAP) / 日内VWAP × 100%
- 偏离度 > +0.5%: 尾盘买方强势，次日高开概率提升
- 偏离度 < -0.5%: 尾盘卖压明显，次日低开概率提升
- 偏离度 ±0.2% 以内: 中性信号
- 结合成交量: 高偏离+高尾盘量比(>5%) = 强信号; 高偏离+低量比 = 弱信号

## 补充分析 (详细展开)

### 被动基金 & ETF 调仓行为特征
- ETF申赎驱动: ETF净申购时，基金管理人需在尾盘竞价买入成分股以最小化跟踪误差
- 指数成分调整: MSCI/富时/中证指数调仓生效日前一天，尾盘竞价成交量可放大5-10倍
- 行为特征: 被动基金的尾盘委托通常是均匀分散的市价单，不会出现极端委买/委卖偏离
- 识别方法: 尾盘竞价量激增但价格偏离度<0.2%，大概率是被动资金(非方向性观点)
- 交易机会: 预判被动资金方向(通过ETF份额变动)，提前布局可获取尾盘冲击的alpha

### 尾盘竞价量 vs 日内成交量比值
- 正常水平: 尾盘竞价量占全天成交量的2-4%，超过5%视为异常
- 高比值含义(>5%): 有机构在尾盘集中下单(调仓/建仓/出货)
- 极端比值(>10%): 通常出现在指数成分调整日或重大事件日
- 低比值含义(<1%): 市场关注度极低，标的流动性差
- 趋势观察: 连续多日尾盘比值上升可能暗示机构持续建仓/出货行为

### 尾盘竞价 vs 开盘竞价: 信号特征差异
- 信息环境不同: 开盘竞价反映隔夜信息消化，尾盘竞价反映日内博弈结果+次日预期
- 参与者不同: 开盘竞价散户参与度高(情绪驱动)，尾盘竞价机构占比更高(理性定价)
- 撤单机制: 开盘有可撤单阶段(9:15-9:20)，尾盘3分钟全程不可撤单，信号纯度更高
- 定价权重: 深市收盘价由尾盘竞价决定(非最后一笔)，收盘价对次日开盘有锚定效应
- 策略差异: 开盘竞价适合做「预期差捕捉」(快进快出)，尾盘竞价适合做「趋势延续确认」(持仓信号)

## 时段状态 & 条件激活机制

### Auction Tab 四种时段状态

| 时段 | 状态名 | 页面展示 |
|---|---|---|
| 9:15-9:30 | 🟢 开盘竞价 | **全面激活**: Card 1 表格实时刷新 + Card 2 异动检测 + 阶段指示器倒计时 |
| 9:30-14:57 | ⚪ 连续交易 | 竞价面板收起, 显示"**今日开盘竞价摘要**"静态回顾 |
| 14:57-15:00 | 🔵 尾盘竞价 | Card 3 尾盘面板激活, 开盘面板保持摘要状态 |
| 15:00+ | 🔴 收盘后 | **全天竞价复盘**: 开盘+尾盘数据归档, 预判准确率统计 |

### 状态切换规则

- 基于系统时钟自动判断当前时段
- 切换时有过渡动画 (面板折叠/展开 + 边框颜色变化)
- 支持浏览器通知: "尾盘竞价即将开始" (14:56 提前1分钟提醒)

### 技术实现要点
- 时钟同步: 使用交易所时间 (非本地时间), 通过 API 服务器同步
- WebSocket 管理: 竞价阶段建立连接, 连续交易阶段断开 (节省资源)
- 数据归档: 每日竞价快照自动存储 (9:15-9:25 每秒 + 14:57-15:00 每秒)
- 状态恢复: 中途刷新页面能根据当前时间自动恢复正确状态
