# Vibe Trading Coding Guidelines

This directory contains the strict coding standards and software design principles for the Vibe Trading project. All contributors must adhere to these guidelines to ensure code quality, maintainability, and scalability.

## Core Pillars

1.  **[Design-First Workflow](./workflow.md)**: Diagrams and interfaces before implementation.
2.  **[Core Design Principles](./principles.md)**: SOLID, SRP, DRY, KISS, YAGNI.
3.  **Language Standards**:
    - [TypeScript & React](./typescript.md)
    - [Python & FastAPI](./python.md)
4.  **Architectural Patterns**:
    - [Design Patterns](./patterns.md) (Strategy, Repository, Factory)
    - [Event-Driven Patterns](./kafka.md) (Kafka, Idempotency)
5.  **[Operational Standards](./ops.md)**: Testing (TDD), Logging, Error Handling.

---

*Refer to these guidelines during the "Analysis" phase of every track.*
