# 订单执行与撮合逻辑 (Execution Logic)

> Vibe Trading 使用**向量化回测引擎**（非事件驱动），参见 backtest.md PRD。

## 1. 订单撮合逻辑

### 参考价格 ($P_{ref}$)

| 类型 | 说明 | 推荐场景 |
|---|---|---|
| 当前收盘价 | 当前 Bar (T) 收盘价立即撮合 | 分钟线/Tick 回测 |
| 下一开盘价 | 下一 Bar (T+1) 开盘价撮合 | **日线回测推荐**，避免未来数据 |
| 最新价 | Tick 流中的实际成交价 | Tick 级仿真 |

### 按订单类型

- **市价单**: 除非停牌/涨跌停，始终以 $P_{ref} \pm \text{滑点}$ 撮合
- **限价单**: 仅当 $P_{ref}$ 满足限价条件时撮合
  - 买入: $P_{limit} \geq P_{ref}$
  - 卖出: $P_{limit} \leq P_{ref}$

## 2. 执行约束

### 成交量限制
- **规则**: 最大成交量 ≤ 当前 Bar 总成交量的 25%
- **市价单**: 部分成交，撤销剩余
- **限价单**: 部分成交，剩余留在队列等待下一 Bar

### 无法成交的情况
- 停牌
- 涨停时买入 / 跌停时卖出（无对手方）
- 资金/保证金不足
- 订单数量小于最小可交易单位（A 股 100 股）

## 3. 市场状态划分 (SVM-GARCH)

### 趋势 vs 震荡
- **价格轨迹效率 (Efficiency)** > 0.25 → 趋势市场
- **效率** < 0.25 → 震荡市场

### 上涨 vs 下跌
- GARCH 模型估计波动率
- SVM (SVC) 对波动率及对冲指标进行分类预测

### 因子分工
| 市场状态 | 侧重模型 | 逻辑 |
|---|---|---|
| 趋势/上涨 | 成长模型 (Growth) | 弹性大 |
| 下跌/震荡 | 价值模型 (Value/SFScore) | 防御性强 |

## 4. 回测偏误

| 偏误 | 起因 | 缓解 |
|---|---|---|
| **未来数据 (Look-Ahead)** | 使用 T 时刻尚未获得的信息（如 T+1 公司行为） | 动态复权——仅应用回测日期前已知的复权因子 |
| **幸存者偏差 (Survivorship)** | 仅测试当前存续证券 | 使用包含每个历史时间点所有活跃股票的动态样本池 |
| **过拟合 (Overfitting)** | 拟合历史噪声而非信号 | 样本内/样本外数据分割 |
| **做空不可行偏误 (Short-Side Bias)** | A 股融券受限，策略做空端收益在现实中无法兑现 | 检查 Robust Universe Return；对信号做 rank→power→right_tail 调整（见 `preprocessing.md` §5）；聚焦因子有效的市值区间 |
